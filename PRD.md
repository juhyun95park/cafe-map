카공여지도

1. Overview
   Problem Statement
   문제 정의: 서울의 20-30대 직장인 및 학생들은 주말이나 퇴근 후 작업/공부할 카페를 찾을 때, 기존 검색 플랫폼(네이버, 구글 등)이 제공하는 '미적(aesthetic)' 정보가 아닌 '실용적(practical)' 정보(콘센트 유무, 조용한 정도 등)를 얻기 어렵습니다. 이로 인해 원하는 카페를 찾기까지 불필요하게 많은 시간과 노력을 소모하는 정보 탐색의 비효율 문제가 발생합니다.
   왜 중요한가: 만족스러운 작업 공간을 찾지 못하는 경험은 개인의 시간 낭비와 감정 소모를 넘어, 생산성 저하로까지 이어집니다. 이 문제를 해결하는 것은 타겟 고객의 일상적인 불편함을 해소하고, 더 가치 있는 곳에 시간을 쓸 수 있도록 돕는 중요한 일입니다.
   타겟 고객:
   연령/직업: 서울 및 수도권에 거주하는 20-30대 직장인, 프리랜서, 대학(원)생
   행동 패턴: 주 2회 이상 카페를 이용하며, 한 번 방문 시 3시간 이상 머무는 '카공족'
   핵심 니즈: 예쁜 인테리어나 유명세보다 조용한 분위기, 충분한 콘센트, 빠른 Wi-Fi 등 작업 환경의 실용성을 최우선으로 고려함
   전략적 적합성: 거대 플랫폼이 모든 사용자를 만족시키기 위해 보편적인 정보를 제공하는 것과 달리, 우리는 '작업/공부에 최적화된 카페'라는 명확한 틈새시장을 공략합니다. 이를 통해 특정 사용자 그룹에게 깊은 가치를 제공하는 하이퍼-로컬(Hyper-local) 서비스로 포지셔닝합니다.
   핵심 전제/가정:
   사용자들은 '예쁜' 카페 정보보다 '실용적인' 카페 정보를 더 절실히 원한다.
   동네 주민들은 자신이 아는 좋은 카페 정보를 다른 이들과 공유할 의향이 있다.
   별점 평가보다 '조용해요', '콘센트 많아요' 같은 구체적인 태그형 정보가 사용자 의사결정에 더 효과적이다.
   Proposed Work
   솔루션 개요: 내 주변 1km 반경 내에서, 동네 주민들이 직접 검증하고 추천한 '작업/공부하기 좋은' 카페 정보를 실용적인 필터와 태그 기반 리뷰를 통해 직관적으로 제공하는 모바일 웹 플랫폼입니다.
   핵심 기능:
   위치 기반 실용 정보 필터링: 현재 위치를 기반으로 주변 카페를 보여주며, '조용한', '콘센트 많은', '주차 가능' 등 실용적인 조건으로 필터링하는 기능
   태그 기반의 실용 리뷰 시스템: 별점 대신 "대화하기 좋아요", "콘센트 자리 많아요", "오래 앉아 있기 좋아요" 등 사용자가 실제 경험을 바탕으로 선택하는 태그형 리뷰
   시간대별 혼잡 경향 표시: 실시간 데이터가 아닌, "주말 오전에 붐벼요", "평일 저녁은 한산해요" 등 유저 리뷰 기반의 혼잡도 경향 정보 제공
   핵심 지역 중심의 카페 DB: 서비스 초기에는 강남, 홍대 등 타겟 고객이 밀집한 지역의 카페 정보를 집중적으로 제공
   사용자 플로우:
   사용자는 서비스에 접속하여 별도 검색 없이 즉시 자기 주변의 카페 목록과 지도를 확인한다.
   '콘센트 많음', '조용함' 등 원하는 조건의 필터를 적용한다.
   필터링된 카페 중 하나를 선택하여 상세 정보(태그 리뷰, 사진, 운영 시간 등)를 확인한다.
   정보를 바탕으로 방문할 카페를 결정하고, 방문 후 직접 경험한 내용을 태그 리뷰로 남긴다.
2. Decision / Risk
   Stakeholders
   직접 이해관계자:
   제품 책임자/개발자 (본인): 기획, 개발, 운영 전반
   초기 사용자 그룹 (리뷰어): 서비스의 초기 콘텐츠를 생성하고 피드백을 제공하는 핵심 기여자
   카페 사장님: 자신의 가게 정보가 등록되고 평가받는 주체 (초기에는 인지하지 못함)
   간접 이해관계자:
   경쟁/대체 서비스: 네이버 지도, 카카오맵, 인스타그램 등
   지역 커뮤니티: 서비스 홍보 및 초기 사용자 유입 채널 (예: 대학생 커뮤니티, 직장인 앱)
   Risk
   주요 위험 요소 1: 초기 콘텐츠 부족 (Cold Start 문제)
   발생 가능성과 영향도: 높음 / 높음
   대응 방안:
   서비스 오픈 전, 지인 및 타겟 커뮤니티를 통해 '초기 리뷰어' 10명 내외를 모집하여 핵심 지역 카페 50개에 대한 리뷰 100개 이상 사전 확보
   운영자가 직접 강남/홍대 지역의 카페를 방문하여 초기 데이터베이스 구축
   주요 위험 요소 2: 데이터의 신뢰성 및 최신성 유지 문제
   발생 가능성과 영향도: 중간 / 높음
   대응 방안:
   MVP 단계에서는 운영자가 한 달 주기로 등록된 카페의 폐업 여부나 정보 변경을 수동으로 확인
   v1.1 업데이트 시 '정보 수정 제안하기' 기능을 추가하여 사용자 참여를 통한 데이터 개선 유도
   주요 위험 요소 3: 사용자 재방문 유도 실패
   발생 가능성과 영향도: 중간 / 높음
   대응 방안:
   신규 카페가 등록되거나 내가 리뷰한 카페에 새로운 태그가 달렸을 때 알림을 주는 기능 고려 (v2)
   '내가 찾은 꿀카페' 등 사용자의 기여 활동을 모아볼 수 있는 간단한 프로필 기능 도입 (v2)
   Dependencies
   외부 서비스: 카카오맵 API (지도 표시 및 위치 기반 검색을 위해 필수적, 무료 플랜 범위 내 사용)
   필요 데이터: 출시 전 강남/홍대 지역 내 카페 최소 50개의 기본 정보(위치, 이름, 운영시간) 및 특징 데이터
   필요 인력: 개발자 1인 (본인), 초기 콘텐츠 생성을 위한 지인 및 커뮤니티 기반의 자발적 참여자 그룹
3. Success Criteria
   핵심 지표 (출시 후 3개월 내 달성 목표)
   지표명 1: 월간 활성 사용자 (MAU)
   측정 방법: Google Analytics를 통해 순 방문자 수 측정
   목표 수치: 300명
   달성 기한: 출시 후 3개월
   지표명 2: 핵심 지역 내 등록 카페 수
   측정 방법: 내부 데이터베이스(DB)에 등록된 강남/홍대 지역 카페 개수 카운트
   목표 수치: 50개
   달성 기한: 출시 후 3개월
   지표명 3: 누적 리뷰(태그) 수
   측정 방법: DB에 쌓인 총 리뷰(태그) 개수 카운트
   목표 수치: 200개
   달성 기한: 출시 후 3개월
   보조 지표
   세션 당 필터 사용률: 사용자들이 얼마나 적극적으로 실용 정보를 탐색하는지 확인
   카페 상세 페이지 조회 수(PV): 사용자의 관심이 실제 행동으로 이어지는지 확인
   리뷰 작성 전환율: 상세 페이지를 본 사용자 중 몇 퍼센트가 리뷰를 남기는지 측정
4. Definition of Done
   필수 조건 (Done)
   위치 기반으로 주변 카페를 지도와 목록으로 보여줄 수 있다.
   조용함, 콘센트 등 최소 3개 이상의 핵심 필터가 정상적으로 작동한다.
   카페 상세 페이지에서 기본 정보와 태그 기반 리뷰를 확인할 수 있다.
   사용자가 신규 태그 리뷰를 남길 수 있다.
   강남/홍대 지역의 카페 50개, 리뷰 100개 데이터가 사전 탑재되어 있어야 한다.
   개선 필요 (but 출시 가능)
   UI/UX 디자인이 투박하고 미적으로 완벽하지 않은 상태
   실시간 혼잡도 표시 기능은 '시간대별 혼잡 경향' 태그로 대체 (예: "주말 오전에 붐벼요")
   별도의 회원가입/로그인 기능 없이, 모든 사용자가 익명으로 리뷰 작성 가능
   치명적 (출시 불가)
   지도 로딩이 5초 이상 걸리거나 실패하는 경우
   필터 적용 시 결과가 나오지 않거나 앱이 멈추는 현상
   리뷰 작성이 DB에 저장되지 않는 문제
5. Nope (하지 않을 것)
   기능명: 회원가입 및 로그인 시스템
   이유: MVP의 핵심은 '정보 탐색' 가치 검증. 회원 시스템은 개발 리소스가 많이 들고, 초기 사용자에게 허들로 작용할 수 있음.
   대안: 로컬 스토리지 등을 활용해 사용자의 필터 선택 기록 등 간단한 정보를 기기에 저장.
   재고려 시점: 사용자 리뷰의 신뢰도 확보 및 개인화 기능(찜하기 등) 도입이 필요해지는 시점 (MAU 1,000명 달성 시).
   기능명: 카페 예약 또는 결제 연동 기능
   이유: 서비스의 핵심 가치와 직접적인 관련이 없으며, 구현 복잡도와 운영 부담이 매우 높음.
   대안: 없음.
   재고려 시점: 장기적인 비즈니스 모델을 구체화하는 단계에서 고려.
   기능명: 커뮤니티, 소셜 기능 (피드, 댓글, 팔로우 등)
   이유: 핵심 가치를 희석시키고, 어뷰징 및 분쟁 관리 등 운영 리소스가 과도하게 발생.
   대안: 없음.
   재고려 시점: MAU 1만 명 이상을 확보하여 강력한 팬덤이 형성된 이후.
6. Post-Launch Plan
   v1.1 (Quick Wins / 출시 후 ~1개월)
   사용자 피드백 기반의 긴급 버그 수정 및 안정성 개선
   사용자들이 가장 많이 요청하는 필터 1~2개 추가
   사용자 요청 기반으로 서비스 지역(예: 성수, 잠실) 소규모 확장
   v2 (Major Features / 다음 분기)
   회원 시스템 도입: 리뷰 신뢰도 강화를 위한 로그인 및 '내 활동' 페이지 추가
   사용자 카페 등록 제안 기능: 사용자가 직접 지도에 없는 카페를 추가 요청할 수 있는 기능 도입
   서비스 지역 공식 확장: 서울 주요 업무/상권 지역으로 서비스 범위 확대
   v3+ (Vision / 장기)
   개인화 추천: 사용자의 필터 선택, 리뷰 성향을 분석하여 '나를 위한 맞춤 카페' 추천 기능 도입
   카테고리 확장: '카페'를 넘어 '조용한 도서관', '작업하기 좋은 공유 오피스' 등 카테고리 확장
   데이터 기반 B2B 솔루션: 축적된 데이터를 바탕으로 특정 상권의 유동인구 특성 분석 등 비즈니스 모델 모색
   최종 목표: **"대한민국에서 작업/공부 공간을 찾는 모든 사람을 위한 필수 플랫폼"**으로 성장.
